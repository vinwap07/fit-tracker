**WorkoutTracker** — это высоконагруженное REST API приложение для спортсменов и любителей фитнеса. Сервис позволяет планировать тренировочные программы, отслеживать прогресс (вес, калории, подходы), валидировать технику выполнения упражнений с помощью видео-модуля и автоматизировать аналитику.

Проект спроектирован с учетом лучших практик enterprise-разработки: масштабируемой архитектуры, фонового выполнения тяжелых задач и кэширования.

---

## 🏗 Архитектура и паттерны

В основе проекта лежат принципы чистого кода и разделения ответственности:
- **DDD:** Бизнес-логика изолирована внутри богатой доменной модели. Выделены агрегаты, сущности и value-объекты, существующие в своем bounded context
- **CQRS:** Архитектурно разделены операции чтения и модификации данных, что оптимизирует производительность БД и избавляет от избыточности DDD-подхода в операциях чтения.
- **Принципы проектирования:** Строгое соблюдение **SOLID**, **KISS** и **DRY**. Код защищен от сильной связанности (high cohesion, loose coupling).

---

## 🚀 Функциональные возможности (MVP+)

- **Управление тренировками:** Конструктор авторских программ (упражнения, подходы, рабочий вес, повторения).
- **Сравнение техники выполнения:** Модуль для загрузки видеозаписей подходов и их визуального сравнения с эталонным исполнением.
- **Интеллектуальная аналитика:** Автоматическое построение графиков изменения веса атлета, объемов тренировок и сожженных калорий.
- **Кастомная база знаний:** Создание уникальных упражнений с текстовыми инструкциями и медиа-доказательствами.
- **Гибкая безопасность:** Ролевая модель доступа (User / Moderator / Admin) на базе ASP.NET Core Identity (JWT-токены + Cookies).

## 🛠 Технологический стек и инфраструктура


| Компонент | Технология / Библиотека | Назначение в проекте |
| :--- | :--- | :--- |
| **Runtime** | .NET 8 / ASP.NET Core | Основной высокопроизводительный фреймворк |
| **Database** | PostgreSQL + EF Core | Хранение реляционных данных, миграции (Code First) |
| **Caching** | Redis | Кэширование профилей пользователей и частых запросов к аналитике |
| **Background Jobs** | Hangfire | Планирование фоновых задач: еженедельный сбор статистики и отправка email |
| **Object Storage** | Minio (S3) | Надежное и изолированное хранение видео тренировок и изображений |
| **Mapping & Validation**| AutoMapper + FluentValidation | Маппинг DTO/сущностей и строгая валидация входящих запросов |
| **Infrastructure** | Docker / Docker Compose | Контейнеризация приложения и всех связанных сервисов (БД, кэш, S3) |
| **Integrations** | SMTP Server / HttpClient | Отправка системных уведомлений и интеграция с внешними API |

## 🧪 Инструкция по локальному запуску

Для запуска проекта вам понадобится установленный **Docker Desktop** и **.NET 8 SDK**.

### 1. Клонирование репозитория
```bash
git clone https://github.com/vinwap07/fit-tracker
cd fit-tracker
```

### 2. Запуск инфраструктуры через Docker Compose
Все зависимости (PostgreSQL, Redis, Minio, Hangfire) настроены и готовы к работе «из коробки»:
```bash
docker-compose up -d
```

### 3. Применение миграций и запуск API
```bash
dotnet ef database update
dotnet run --project WorkoutTracker.WebApi
```
После этого API станет доступно по адресу `http://localhost:5000`. Интерактивная документация **Swagger** откроется автоматически по адресу `http://localhost:5000/swagger`.
